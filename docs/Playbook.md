# FanAtlas Playbook

**Version:** 0.1  
**Status:** Draft  
**Owner:** FanAtlas Team  
**Last Updated:** 2026-07-03

---

# 1. Purpose

FanAtlas exists to preserve the memories of real-life encounters between sports fans around the world.

Technology is the enabler.

People are the purpose.

---

# 2. Vision

Build the world's most authentic collection of sports fan encounters.

Every profile tells the story of a real person, met in the real world.

---

# 3. Product Principles

## KIS

Keep It Simple.

Every new feature must justify its existence.

If removing a feature improves the product, remove it.

---

## Power of Concision

Collect only information that creates value.

Ask as few questions as possible.

Show only what matters.

---

## Authenticity

Every profile represents a genuine, in-person encounter.

No imported contacts.

No scraped data.

No fake profiles.

---

## Memories, not Metrics

The goal is not engagement.

The goal is preserving meaningful memories.

---

## Privacy

Respect every person's privacy.

Only publish information with the person's consent.

Users must be able to request edits or deletion of their profile.

---

# 4. Blue Moon Scope

The first release focuses on one Atlas only:

**Citizen Fan Atlas**

Supported club:

- Manchester City

Every profile includes:

- One photograph
- Name
- City
- Country
- GPS location
- Date met (automatic)
- City Story
- Public URL
- Atlas ID

---

# 5. User Experience Principles

The map is the heart of the application.

The list complements the map.

Navigation should feel effortless.

The interface should never distract from the stories.

---

# 6. Technology Principles

Prefer open standards.

Prefer simple solutions.

Avoid unnecessary dependencies.

Choose technologies that are easy to maintain.

Build for reliability before scale.

---

# 7. Documentation Principles

If it isn't documented, it doesn't exist.

Documentation should explain:

- Why
- What

Code explains:

- How

---

# 8. Decision Process

Important product decisions are recorded in:

docs/Decisions.md

Every decision receives:

- ID
- Date
- Description
- Reason
- Status

---

# 9. Success

Blue Moon is successful when a user can:

1. Meet a fan.
2. Add the profile in less than two minutes.
3. See that fan immediately appear on the map.
4. Share the public profile.

Nothing else matters until these four steps work beautifully.

---

# 10. Future

FanAtlas is designed to support multiple Atlases.

Examples:

- Citizen Fan Atlas
- Boston Celtics Fan Atlas
- Formula 1 Fan Atlas

Expansion begins only after Blue Moon is complete.

---

# Non-Negotiables

✓ KIS

✓ Power of Concision

✓ One real encounter = one profile

✓ One photo per profile

✓ Privacy first

✓ English for Blue Moon

✓ The map is the hero

✓ Build for memories, not metrics
